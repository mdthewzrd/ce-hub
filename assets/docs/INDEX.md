## üìã **Document Management**

### File Organization Principles
1. **Canonical docs** remain in `/docs/` root
2. **Project-specific** documents in `/docs/projects/{project}/`
3. **Working files** in `/docs/working/`
4. **Archived materials** in `/docs/archived/`

### Maintenance Guidelines
- Update this index when adding new canonical documents
- Move completed working documents to project directories
- Archive deprecated documents regularly
- Maintain consistent naming conventions

### Quick Navigation Commands
```bash
# Find documentation
find docs -name "*keyword*.md"

# Search content
grep -r "search term" docs/

# List project documents
ls docs/projects/*/
```
