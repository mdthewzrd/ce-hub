# 🏁 Traderra — Primary Product Requirements Document (Master v1.0)

## 1) Overview
Traderra is a **professional trading performance and journaling platform** centered around Renata — an AI analyst designed to deliver insight, structure, and accountability for traders. The platform combines automated analytics, intelligent journaling, and adaptive personalization in a cohesive, visually refined interface.

This master PRD aggregates all domain-level documents and defines the cross-functional requirements linking them.

---

## 2) Core Product Vision
> “A professional, AI-driven environment where traders understand, refine, and scale their performance — truthfully and without noise.”

Traderra emphasizes:
- Precision over gamification.
- Data integrity and truthful feedback.
- Visual calm and functional clarity.
- Seamless onboarding and immediate usability.
- Adaptability for each trader’s style without complexity.

---

## 3) Platform Pillars
1. **Performance Analytics:** Data-driven dashboards with KPIs: PnL, Win%, Profit Factor, Expectancy, Avg R/Winner.
2. **AI Journaling:** Structured + freeform journals, linked to trades and metrics.
3. **Reviews & Insights:** Automated daily/weekly/monthly performance reports generated by Renata.
4. **Integrations:** Broker data import, CSV handling, Notion/Drive sync.
5. **Adaptive Interface:** Toggle modes (% | $ | R), gross/net basis, customizable layouts.
6. **Professional UX:** Dark Studio theme with off-whites and gold accents.

## 5) Functional Requirements
### 5.1 Core User Journeys
- **Onboarding:** AI-guided; connects broker/CSV, sets preferences, calibrates tone.
- **Daily Workflow:** User logs trades → Renata contextualizes → daily review draft auto-generated.
- **Weekly Review:** AI compiles summary, charts, and commentary.
- **Customization:** Users toggle stat units, choose AI mode, and adjust layout widgets.

### 5.2 Data Integrity
- CSV imports auto-map formats from major brokers.
- All calculations server-side, double-precision validated.
- KPI formulas documented in Analytics Spec.

### 5.3 AI Behavior
- Adaptive tone within professionalism spectrum.
- Always data-grounded; never motivational or speculative.
- Learns context through workspace memory embeddings.

### 5.4 Sync & Export
- One-click Notion/Drive sync.
- Exports (CSV/ZIP) respect gross/net and %/$/R states.

### 5.5 Security
- RLS enforced globally.
- OAuth credentials encrypted.
- PII redaction before embeddings.

## 8) Design Summary
**Theme:** Dark Studio  
**Primary Colors:** Deep gray/black (#0D0D0D–#121212), Off-white (#EDECEA), Gold (#CFAE60) accents.  
**Typography:** Inter / Space Grotesk, 14–18px base, 32px headers.  
**Motion:** Soft fade/slide; no bounce.  
**Focus:** Readability, breathing space, professionalism.

---

## 9) Technical Architecture
- **Frontend:** Next.js + Tailwind + CopilotKit (AGUI) for state linking.
- **Backend:** FastAPI + PydanticAI + PostgreSQL (pgvector) + Redis.
- **Workers:** Celery / RQ for async jobs (imports, reviews, syncs).
- **Storage:** S3 + presigned uploads.
- **AI Runtime:** Local orchestration layer w/ OpenAI API backend.
- **Integrations:** Polygon, Notion, Google Drive, Clerk.

## 11) Open Risks & Mitigation
| Risk | Mitigation |
|------|-------------|
| Over-customization complexity | Keep onboarding deterministic; mode-based variation only |
| Model output inconsistency | Rule validators + snapshot tests |
| API rate limiting from providers | Queue backoff and token rotation |
| Long-term embedding cost | Vector compression + TTL rotation |
