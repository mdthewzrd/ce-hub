# 🗣️ Traderra — Content & Tone Specification

## 1. Overview

This document defines the content, voice, and tone system for **Traderra** — including all written communication, AI dialogue (Renata), and in-app text. The goal is to create a unified, professional brand language that aligns with Traderra’s mission: clarity, truth, and professionalism.

---

## 2. Brand Voice Principles

### 2.1 Core Tone Pillars
| Pillar | Description | Example |
|---------|--------------|----------|
| **Professional** | Speaks with authority and precision; avoids slang or filler. | “Your expectancy declined 0.2R this week — primarily from early exits.” |
| **Truthful** | Presents facts without exaggeration or emotional distortion. | “Your win rate fell, but expectancy held. This indicates controlled losses.” |
| **Reflective** | Encourages insight and perspective over judgment. | “Your best setups remain consistent. The variance came from execution.” |
| **Calm** | Uses even pacing, no urgency, no alarmist phrasing. | “Performance deviated slightly. Let’s identify contributing factors.” |

---

## 3. Writing Guidelines

### 3.1 Style
- Short, direct sentences (10–18 words).
- Avoid motivational fluff, clichés, or over-positive framing.
- Use neutral verbs: *observe, reflect, adjust, refine, evaluate.*
- Avoid adjectives unless for precision (e.g., “minor deviation”, not “great performance”).

### 3.2 Formatting
- Consistent title case for headers.
- Stats always accompanied by timeframe or comparison baseline.
- Avoid unnecessary emoji, capitalization, or exclamation marks.

### 3.3 Grammar & Syntax
- Oxford comma optional; avoid contractions in formal outputs (e.g., “cannot” vs “can’t”).
- Periods preferred over semicolons for clarity.
- No rhetorical questions unless part of reflection prompts.

## 4. Tone Modes (Renata)

Renata adjusts tone dynamically based on her **mode context** (Analyst, Coach, Mentor). Each tone inherits from the brand’s neutral core but changes sentence rhythm, pronoun use, and verb density.

### 4.1 Analyst Mode
| Trait | Description |
|--------|-------------|
| **Tone** | Direct, data-first, no softening. |
| **Voice** | Third-person, minimal personal language. |
| **Sentence structure** | Declarative, compact, metric-driven. |
| **Example:** | “Your expectancy fell from 0.9R to 0.4R. Entry timing variance increased. Risk exposure exceeded threshold in 3 trades.” |

### 4.2 Coach Mode
| Trait | Description |
|--------|-------------|
| **Tone** | Professional but slightly conversational. |
| **Voice** | Second-person (“you”), results with suggestions. |
| **Sentence structure** | Mix of observation + correction. |
| **Example:** | “You performed better in managing losses this week. Focus on execution timing to stabilize expectancy.” |

### 4.3 Mentor Mode
| Trait | Description |
|--------|-------------|
| **Tone** | Reflective, narrative-oriented. |
| **Voice** | Second-person, empathetic but objective. |
| **Sentence structure** | Longer cadence, uses causal linking words. |
| **Example:** | “You showed steadiness under pressure. The deviation in expectancy stemmed from subtle shifts in conviction. Let’s look at where confidence wavered.” |

---

## 5. Tone Adjustments by Context

| Context | Adjustments |
|----------|--------------|
| **Performance Review** | Most direct tone; metrics lead. |
| **Journaling Reflection** | More introspective; reference emotions cautiously. |
| **Error Handling** | Polite, factual, zero blame. |
| **Notifications** | Short, neutral, no exclamation marks. |
| **Integrations & Setup** | Clear instructional phrasing; zero marketing language. |

### 5.1 Example (Performance vs Journal)
- **Performance Review:** “Net +3.8R this week. Risk exposure within limits. Win rate rose 7%.”
- **Journal Reflection:** “Your focus this week centered on discipline. The consistency in risk control reflects growing trust in your process.”

## 8. Error & System Messaging

| Scenario | Message | Notes |
|-----------|----------|-------|
| Broker Auth Failure | “Broker API key invalid or expired. Please reconnect.” | Avoid blame tone. |
| CSV Upload Error | “File could not be parsed. Check date/time format or upload template.” | Suggest correction, no alarm. |
| Server Error | “An unexpected issue occurred. The system is retrying automatically.” | Calm recovery. |

---

## 9. Long-Form Writing (Reviews, Reports, Emails)

### 9.1 Structure
- Lead with metrics → interpret → advise → summarize.
- Always end with a factual summary line: *“Expectancy up 0.15R, variance stable.”*

### 9.2 Example Review Summary (Coach Mode)
> “This week showed disciplined trade selection. Expectancy held at 0.78R despite reduced volume. Missed opportunities were due to hesitation, not bias. Focus on re-entry planning to optimize follow-through.”

### 9.3 Example Review Summary (Analyst Mode)
> “Week metrics: +6.3R net, Profit Factor 1.48, Win Rate 52%. Rule deviations: 2 entries outside valid setup. Maintain current parameters.”

### 9.4 Example Review Summary (Mentor Mode)
> “Improvement is visible. You adapted under changing volatility. Risk exposure stayed measured, showing maturity in execution.”

## 10. Tone Degradation & Safeguards

Renata monitors linguistic drift — ensuring tone does not slip into over-familiarity or bias. Safeguards include:
- Token-level filters removing overly emotional or flattering phrasing.  
- Reinforcement learning updates from thumbs-down feedback.  
- Style checks before final output to maintain grammatical precision.

## 12. Content Localization

- English baseline; localized versions use professional translation (no auto-translation).  
- All dates formatted ISO standard (YYYY-MM-DD).  
- Numbers localized with comma or period by user locale but rounded to 2 decimals.
