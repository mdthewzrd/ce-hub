# 📊 Traderra — Analytics & Visualization Specification

## 1. Purpose
Defines **all analytics, KPIs, chart types, unit toggles, aggregation rules, and interpretation logic** used across Traderra. This is implementation-grade and aligned with the Data & Integration spec. Renata relies on these definitions for truthful, consistent narratives.

---

## 2. Global Controls & Context

### 2.1 Gross/Net Toggle
- **Global state:** `pnl_basis ∈ {gross, net}`.
- **Effect:** Applies to **all** KPIs, charts, tooltips, and exports. Net deducts fees/commissions/slippage when present.

### 2.2 Unit Toggle: `% | $ | R`
- **Global state:** `unit_mode ∈ {percent, absolute, r_multiple}`.
- **Transform:**
  - `%` — values shown as percent relative to selected baseline (P&L vs prior period or equity, see below).
  - `$` — absolute currency in user account currency.
  - `R` — normalized by user-configured risk per trade (`R = (ExitValue - EntryValue)/RiskAmount`).
- **Propagation:** Tiles, charts axes, table cells, exports, and Renata’s phrasing.

### 2.3 Filters & Aggregation Window
- Default: **Last 30 days | All accounts | All instruments**.
- Supported filters: `date_range`, `symbols[]`, `accounts[]`, `strategies[]`, `tags[]`, `session (RTH/ETH)`, `side (long/short)`.
- Time bucket granularity auto-select:
  - ≤ 45d → daily
  - ≤ 18mo → weekly
  - > 18mo → monthly

## 3. KPI Definitions (Canonical)

> All KPIs must specify basis (gross/net), unit mode, and range. Round to 2 decimals by default.

1) **Gross/Net P&L**  
`sum(RealizedPnL)` (net subtracts fees). If unit `%`, express as % of **starting equity** for range.

2) **Win Rate (WR)**  
`#WinningTrades / #ClosedTrades`.

3) **Profit Factor (PF)**  
`Sum(GrossProfits) / Sum(GrossLosses)`; losses absolute value in denominator.

4) **Expectancy (E)**  
`E = (WR * AvgWin) - ((1 - WR) * AvgLoss)`  
Respect unit mode: in `$` or `R`.

5) **Avg R per Winner / Loser**  
`Avg(R | R > 0)` and `Avg(R | R < 0)`.

6) **Average Trade Duration**  
`mean(closed_at - opened_at)`; show winners/losers separately when relevant.

7) **Max Drawdown (MDD)**  
From equity curve: `max(peak - trough)/peak` (if `%`) or absolute delta (if `$`), or in `R` using normalized equity.

8) **Sharpe (optional, informational)**  
`(mean(returns) - rf) / std(returns)`; periodization clarified in tooltip. Use daily returns; rf ≈ 0 by default.

9) **SQN (System Quality Number)**  
`SQN = mean(R) / std(R) * sqrt(n)`; `n >= 30` recommended.

10) **MAE / MFE**  
Per trade, min/max excursion against/with position before close; aggregated mean by setup/instrument.

11) **Fees/Commissions**  
Sum of `executions.fee` in range.

12) **Strategy Compliance (%)**  
`1 - (#RuleViolations / #RuleChecks)`; rules sourced from `strategies.rules` if user-defined.


**Notes:**
- All averages exclude NaN/inf; PF undefined if no losses — display `—`.
- Rounding: tiles 2dp; tooltips 4dp if needed.

## 4. Chart Library & Specifications

### 4.1 Equity Curve (Cumulative P&L)
- **X:** time bucket; **Y:** cumulative P&L (basis + unit).
- Source: aggregated trades; include open P&L only if explicitly toggled (v2+).
- Visual: single smooth line; subtle gold glow; hover reveals bucket stats.
- **Annotations:** new high, drawdown start/end.

### 4.2 Daily/Periodic P&L Bars
- **X:** date; **Y:** P&L (basis + unit).
- Tooltip: date, trades count, win rate, total R.

### 4.3 Distribution: R-Multiple Histogram
- **X:** R bins (e.g., -3R .. +5R); **Y:** frequency.
- Overlay: mean R line; shaded 1σ band.

### 4.4 Expectancy by Instrument
- Bar/column chart: instrument on X, expectancy on Y (unit-aware).
- Sort: descending expectancy.

### 4.5 Performance by Weekday / Hour
- Heatmap or bar set:
  - Weekday (Mon–Fri) vs Expectancy/Win%.
  - Hour-of-day buckets (local tz or market tz) vs Expectancy.

### 4.6 Drawdown Curve
- From equity high-water mark; Y negative down to max drawdown.
- Mark deepest trough and recovery length.

### 4.7 Duration Distribution
- Histogram of trade durations; split winners vs losers.

### 4.8 Strategy Compliance Timeline
- Time series of compliance %; overlay rule-violation markers.

### 4.9 MAE/MFE Scatter
- Scatter per trade: X=MAE, Y=MFE (unit-aware); color by result sign.

### 4.10 Setup Cohorts
- Small multiples or grouped bars: expectancy by declared setup/tag.

## 5. Data Marts & Queries (API-Level)

### 5.1 Aggregation Contracts
- **Endpoint:** `/api/stats/summary?range=&filters=` → tiles (WR, PF, E, Avg R, Fees, MDD, etc.)
- **Endpoint:** `/api/stats/equity?range=&bucket=` → cumulative series
- **Endpoint:** `/api/stats/pnl_bars?range=&bucket=` → daily/weekly bars
- **Endpoint:** `/api/stats/distribution?r_bins=` → R histogram
- **Endpoint:** `/api/stats/grouped?by=instrument|weekday|hour|setup` → grouped metrics

### 5.2 Example SQL Patterns (conceptual)
- Equity: `SELECT bucket, SUM(pnl_basis) OVER (ORDER BY bucket) AS cum FROM pnl_agg WHERE ...`  
- PF: `SUM(GREATEST(pnl_basis,0)) / NULLIF(ABS(SUM(LEAST(pnl_basis,0))),0)`
- Expectancy: `wr*avg_win - (1-wr)*abs(avg_loss)` where `wr = wins/closed`

### 5.3 Caching
- Hot window (≤ 90d) cached in Redis keyed by `(workspace, filters, unit, basis)`.
- Cold data cached in Postgres materialized views refreshed nightly + on-demand.

## 6. Interpretation Rules (Renata)

Renata converts analytics into narratives using deterministic rules + thresholds.

### 6.1 Delta Narratives
- If `ΔE ≥ +0.2R` week-over-week → “Expectancy improved; investigate the drivers.”
- If `ΔWR ≥ +5%` but `E ↓` → “Win rate up but expectancy down; risk/reward imbalance likely.”
- If `PF > 1.6` and `MDD shrinking` → “System stability improving.”

### 6.2 Anomaly Detection
- Z-score on key metrics vs 12-week baseline.  
- Flag if `|z| ≥ 2` (medium), `|z| ≥ 3` (high).  
- Outlier trades: `|R| ≥ 3σ` or rule violations.

### 6.3 Rule-Driven Insights
- Strategy rules from `strategies.rules` produce compliance checks per trade.
- Non-compliant clusters → “Most deviations on Tuesdays during open hour.”

### 6.4 Unit-Aware Phrasing
- `%` → “up 6.3% vs prior period.”
- `$` → “+$4,520 net.”
- `R` → “+5.7R cumulative.”

### 6.5 Tone Mapping
- Analyst → report deltas concisely.  
- Coach → add corrective suggestion.  
- Mentor → correlate with journal sentiment (from embeddings).

## 7. Accessibility & Readability
- High-contrast charts; grid lines soft gray; labels off-white.
- Color semantics muted (no aggressive red/green).  
- Tooltip content ≤ 3 metrics + 1 insight.
- Keyboard support: focusable data points; chart zoom reset via keyboard.

---

## 8. Edge Cases & Fallbacks
- No trades: render educational empty states; Renata offers import help.
- PF undefined: show `—`; tooltip: “No losses in range.”
- Small sample size `n < 20`: display caution banner on SQN and Sharpe.
- Mixed currency accounts (future): normalize via FX rates; v1 assumes single currency.

---

## 9. Export Rules
- Exports inherit current `pnl_basis` and `unit_mode`.
- CSV headers include `basis`, `unit`, `range`, `filters` metadata rows.
- PDF charts include axis labels consistent with unit mode.
