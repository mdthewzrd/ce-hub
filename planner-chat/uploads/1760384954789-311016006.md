# 🔗 Traderra — Notion & Drive Sync Specification (v1)

## 1) Purpose
Defines how Traderra synchronizes journals, reviews, and templates with **Notion** and **Google Drive**, maintaining structural fidelity, metadata parity, and audit safety. Supports one-way (push) sync in v1, with bi-directional (pull + conflict resolution) planned for v2.

---

## 2) Scope
- Journals, reviews, templates.
- Workspace-level integration credentials (OAuth2 per provider).
- Metadata synchronization: title, last edited, author, tags, and document ID.
- Sync modes: manual push, auto-sync (per workspace toggle).
- Export format: Markdown → Notion blocks or Google Docs API structure.

---

## 3) Architecture Overview
| Layer | Description |
|-------|-------------|
| Traderra Backend | Generates Markdown/JSON representation. Handles OAuth, rate limits, retries. |
| Sync Service (Worker) | Queued jobs for each document push; idempotent. |
| Provider Adapter | Notion SDK / Google Drive SDK wrappers with deterministic mapping. |
| Audit Layer | Logs sync job ID, source doc ID, remote ID, checksum, timestamps. |

---

## 4) OAuth & Auth Handling
- OAuth2 via Clerk → Brokered through backend (no client secret on frontend).
- Store only refresh tokens (AES-256-GCM encrypted).
- Scopes:
  - **Notion:** `pages:read`, `pages:write`, `databases:read`.
  - **Drive:** `drive.file` (per-user file scope only).
- Token rotation on expiry; revoke on disconnect.

---

## 5) Sync Flow (Push)
1. User triggers sync → `/v1/integrations/{provider}/sync`.
2. Job queued with `{workspace_id, document_id, provider}`.
3. Worker fetches document (Markdown + JSON blocks).
4. Adapter maps Markdown → provider format.
5. Upload/create or update remote object.
6. Store mapping `{document_id, provider, remote_id, last_synced_at, checksum}`.

**Response:** `{ "sync_id": "uuid", "status": "queued" }`

---

## 6) Document Mapping

### 6.1 Common Fields
| Field | Traderra | Notion | Google Drive |
|--------|-----------|--------|---------------|
| Title | `title` | Page title | Document title |
| Tags | `tags[]` | Multi-select property | Metadata property `traderra_tags` |
| Date | `created_at`, `updated_at` | Created/Last edited | File metadata |
| Author | `user_id` | Mention property | File owner |
| Type | `kind` | Database column (template type) | None (custom field) |

### 6.2 Content Blocks
#### Traderra JSON → Notion Blocks
| Traderra Node Type | Notion Block | Mapping Details |
|--------------------|---------------|-----------------|
| heading | heading_1/2/3 | Preserve level |
| paragraph | paragraph | Rich text annotated |
| list_item | bulleted_list_item | Ordered detection by numeric prefix |
| table | table | Convert rows + cells |
| chart_ref | toggle_block (collapsed) | Embedded preview link |
| stat_summary | callout block | Styled summary |
| code | code block | Language autodetect |
| divider | divider | Direct map |

#### Traderra JSON → Google Docs
| Traderra Node Type | Google Docs Element | Notes |
|--------------------|---------------------|-------|
| heading | Paragraph style H1–H3 | Style map |
| paragraph | Paragraph | Preserve bold/italic/links |
| list_item | Bullet | Numeric auto-detect |
| table | Table | Fixed widths |
| code | Paragraph + monospace font | No syntax color |
| divider | HorizontalRule | Simple break |

## 7) Conflict Handling (v2)
### 7.1 Detection
- Compare remote `last_edited_time` vs local `updated_at`.
- If remote newer → mark conflict.

### 7.2 Resolution Modes
| Mode | Description |
|------|-------------|
| **Last-Writer-Wins (Default)** | Preserve most recently updated. |
| **Prompt User** | Ask via Renata if overwrite is safe. |
| **Merge (Experimental)** | Block-level merge using MD5 checksums per block. |

### 7.3 Audit Entry Example
```json
{
  "job_id": "uuid",
  "document_id": "uuid",
  "provider": "notion",
  "remote_id": "abc123",
  "action": "update",
  "status": "success",
  "checksum_local": "sha256...",
  "checksum_remote": "sha256...",
  "conflict": false,
  "duration_ms": 820
}
```

## 8) Error Handling & Retries
| Error | Retry Policy | User Message |
|--------|---------------|---------------|
| 401 Unauthorized | Refresh token + retry | “Your integration expired. Please reconnect.” |
| 403 Permission | Fail | “Missing permission. Adjust integration scopes.” |
| 404 Not Found | Remove mapping | “Remote page deleted. Re-sync to recreate.” |
| 429 Rate Limit | Exponential backoff (2, 4, 8, 16s) | Silent retry |
| 500 Provider Error | Max 3 retries | Logged, notify user if fails persistently |

## 9) Metadata & Audit Storage
Table: `integration_sync_logs`
| Column | Type | Description |
|---------|------|--------------|
| id | UUID | Job ID |
| workspace_id | UUID | Tenant scope |
| document_id | UUID | Traderra doc |
| provider | ENUM(notio, drive) | Integration source |
| remote_id | TEXT | External page/file ID |
| checksum | TEXT | SHA256 content hash |
| action | ENUM(create, update) | Operation performed |
| status | ENUM(queued, running, success, failed) | Job status |
| duration_ms | INT | Job runtime |
| error_message | TEXT | Optional failure note |

---

## 10) Rate Limits & Quotas
| Provider | Limit | Traderra Behavior |
|-----------|--------|--------------------|
| Notion | 3 req/sec | Queue burst, concurrency control 1/job |
| Drive | 10 req/sec/user | Batching, backoff |
| Worker | 50 concurrent jobs max | FIFO priority queue |

---

## 11) Sync Settings Schema
```json
{
  "auto_sync": true,
  "providers": {
    "notion": {"enabled": true, "default_database_id": "abc123"},
    "drive": {"enabled": false}
  }

```
Stored under `workspace_preferences.integrations`.

---

## 12) Notifications
- `sync.completed` WebSocket event → `{provider, document_id, status}`
- Renata may reference sync events contextually: “Your daily review was synced to Notion successfully.”

---

## 13) Security
- Tokens encrypted AES-256-GCM, key rotation 90 days.
- No token exposure in logs or client responses.
- Sync payloads sanitized (no raw PII, broker IDs, or internal metrics).
- All outbound API calls signed + traced (OpenTelemetry span `integration.sync`).

---
