# ✅ Traderra — Test Plan & QA Strategy (v1)

## 1) Purpose
Define a comprehensive **test strategy, cases, fixtures, tooling, and acceptance criteria** for Traderra’s MVP→Launch scope. Covers **unit, integration, E2E**, load, security, accessibility, and AI evaluation with deterministic harnesses.

---

## 2) Scope & Objectives
- Verify correctness of **imports, analytics, toggles (%/$/R, Gross/Net)**, journaling, reviews, and Renata flows.
- Validate **multi-tenant security** (RLS), OAuth token handling, and audit trails.
- Ensure **performance SLAs** (API latency, AI sync) and **usability** (readability, keyboard nav).
- Establish **repeatable fixtures** and **golden snapshots** for analytics & AI text outputs.

---

## 4) Tooling

## 7) Detailed Test Cases

### 7.1 Authentication & Tenancy
- **RLS Enforcement**: Create data in workspace A; assert non-visibility from workspace B.
- **JWT Expiry**: Expired token → 401; refresh flow.
- **Role Matrix**: `owner/admin/member/viewer` permissions on mutating endpoints.

### 7.2 CSV Imports
- **Header Mapping**: Auto-detect column maps for IBKR, Tradervue, Edgewonk; allow manual override.
- **Timezone Inference**: Missing tz → infer from `trade_accounts.market_tz`; assert UTC storage.
- **Idempotency**: Re-apply same batch; no duplicate trades (hash key).
- **Validation Errors**: Negative qty + `side=buy` flagged; recovery prompt.
- **Partial Apply**: 98% success; 2% rejected with reasons; transaction boundaries validated.

### 7.3 Broker API Pulls (Phase 2+)
- **Rate Limit**: Simulate 429; exponential backoff + jitter.
- **Token Refresh**: 401 → refresh → retry success.
- **Duplicate Handling**: External IDs conflict; upsert not duplicate.

### 7.4 Trades & Executions
- **Exec Ordering**: FIFO by timestamp; rounding stable to 1e-6.
- **PnL Calc**: Matches reference spreadsheet to 0.01.
- **Outlier Tagging**: |R| > 3σ auto-flag; UI badge present.

### 7.5 Market Data Cache
- **Write-Through**: Cache miss → fetch Polygon → persist → serve.
- **Backfill Job**: Partitioned pulls; idempotent chunks; BRIN performance.
- **Stale Data Banner**: If cache older than threshold.

### 7.6 KPIs & Analytics
- **PF Undefined**: No losses → ‘—’ with tooltip.
- **Expectancy**: Check transformation across `%/$/R`.
- **Equity Curve**: Sum of daily bars equals cumulative delta.
- **Drawdown**: Synthetic series verify max trough; chart labeling.
- **Distribution**: Histogram bins stable with seed; mean/σ overlays accurate.

### 7.7 Global Toggles
- **Gross/Net**: All tiles/charts/tooltips switch; export inherits basis.
- **Unit `%/$/R`**: Axes labels, tiles, tooltips, *and AI phrasing* update.
- **Persistence**: Toggle states saved per-user; survive reload.

### 7.8 Journaling
- **Template Selection**: Structured vs freeform; autosave drafts.
- **AI Tagging**: “hesitation”, “early exit” → creates semantic tags; embedding persisted.
- **Linkage**: Journal ↔ trades ↔ reviews maintained; search returns by sentiment.

### 7.9 Reviews
- **Daily**: Correct aggregation; narrative references actual deltas.
- **Weekly**: Multi-day data; compare to previous week; anomaly flags present.
- **PDF Export**: Charts render; units consistent with toggles.

### 7.10 AI Queries (Renata)
- **Mode Tones**: Analyst vs Coach vs Mentor style snapshots; rule-based validator checks tone keywords.
- **Unit Awareness**: “Switch to R” → response values and wording reflect R.
- **Context Link**: On dashboard, Renata reads filters and date range.
- **Tool Calls**: `analyze_performance` called with correct args; retries on failure.

### 7.11 Notion/Drive Sync
- **Push-Only**: Mapping for headings, paragraphs, toggles, tables; idempotent if no change.
- **Conflict (v2)**: Last-writer-wins per block; audit trail entries.

### 7.12 Exports
- **CSV**: Headers include metadata rows (basis, unit, filters).
- **ZIP**: Contains journals (MD+JSON), trades CSV, reviews PDF.
- **Signed URL**: TTL expiration; 403 after expiry.

### 7.13 WebSockets
- **Auth**: Reject unauthenticated; accept with JWT + workspace.
- **Events**: `ai.message`, `review.completed`, `import.progress`, `stats.updated` flow through.
- **Reconnect**: Exponential backoff; message replay where applicable.

### 7.14 Accessibility (A11y)
- **Contrast**: Verify 7:1 ratios on key text.
- **Keyboard**: Tab order, focus rings, and shortcuts.
- **Screen Reader**: ARIA labels for charts, alt text for icons/buttons.

### 7.15 Internationalization
- **Locale Numbers**: Thousands separators; decimals; `%` properly placed.
- **Dates**: ISO display with locale option; consistent parsing.

## 8) Performance Tests

### 8.1 API
- P50 < 300ms, P95 < 800ms across `/stats/*`, `/trades`, `/documents`.
- Burst tests at 10x sustained QPS; no error >2%.

### 8.2 Workers
- Imports: 50k rows < 90s; memory bound < 512MB.
- Review jobs: 95th percentile < 15s.

### 8.3 WebSocket
- Round-trip latency < 200ms for AI context on LAN; < 400ms WAN.

### 8.4 Frontend
- First meaningful paint < 2s on mid-tier laptop; bundle < 300KB critical path.

---

## 9) Security Tests
- **RLS**: Attempt cross-workspace data access → denied.
- **Token Storage**: Ensure encryption at rest; cannot retrieve raw tokens via API.
- **OWASP Top 10**: XSS (React-escaped), CSRF (JWT+same-site), SSRF prevention on webhooks.
- **Headers**: HSTS, CSP (script-src self), X-Content-Type-Options.
- **Secrets**: gitleaks scan passes; no plaintext keys in repo.

---

## 10) Acceptance Criteria (MVP)
- User completes onboarding in ≤ 10 minutes with Renata.
- CSV import of 1k+ trades succeeds with ≤ 1% error after mapping.
- Dashboard shows five KPIs, equity curve, daily P&L, and recent journals with correct toggles.
- Daily/Weekly reviews generate accurate KPIs; narratives align with data.
- Gross/Net and %/$/R toggles apply globally and persist per-user.
- Exports contain correct metadata, respect toggles, and download via signed URLs.
- A11y: Keyboard navigable; essential charts accessible.

---

## 11) QA Process & Reporting
- **Branch PR checks**: unit + API contract + lint.
- **Nightly E2E**: Playwright against staging with fresh seeds.
- **Weekly Perf**: k6 run; regressions auto-filed.
- **Release Gate**: All P0/P1 issues closed; test coverage ≥ 80% on core services.

---

## 12) AI Evaluation Harness
- Golden outputs for 30 canonical prompts per mode (Analyst/Coach/Mentor).
- Rule validators: forbid coddling phrases; enforce unit- and basis-aware wording.
- Drift monitor: cosine similarity to style vectors; alert if > 2σ deviation.

---

## 13) Risk Register & Mitigations
- **Inconsistent CSV schemas** → Interactive mapper + saved profiles.
- **Polygon outages** → Serve cached; mark data as stale.
- **High variance in AI outputs** → Snapshot + rule validators + user override controls.
- **Performance spikes on reviews** → Queue backpressure + batch windowing.
