# 🤖 Traderra — AI Architecture Document (Renata System)

## 1. Overview

**Renata** is the central AI orchestrator of the Traderra ecosystem. She embodies adaptive professionalism — a single intelligent system capable of analyzing trade data, guiding reflection, and adapting her tone and behavior based on user context.

Renata operates through three distinct modes of professional engagement — **Analyst**, **Coach**, and **Mentor** — each mapping to a contextual reasoning profile and tone framework. All modes share a unified memory system, knowledge base, and contextual graph.

The AI stack is fully integrated into Traderra’s architecture using **AGUI**, **Co-PilotKit**, **FastAPI**, **pgvector**, and **Redis** for efficient state streaming and retrieval.

---

## 2. Core Architecture

### 2.1 System Layers

1. **Frontend Layer (AGUI + Co-PilotKit)**  
   - Provides contextual state awareness between user interface and AI runtime.  
   - Syncs active UI elements (journal, trade log, dashboard filters) to Renata’s context graph.  
   - Streams updates in <200ms latency using WebSockets.

2. **Middleware / API Layer (FastAPI)**  
   - Manages API endpoints for AI queries, user data access, and analytics aggregation.  
   - Handles requests from frontend via secure JWT-authenticated sessions.  
   - Integrates with Renata Runtime for context packaging.

3. **Renata Runtime (PydanticAI)**  
   - The orchestration layer where Renata’s reasoning, tone adaptation, and tool invocation occur.  
   - Includes submodules for trade analysis, journaling insight, and data correlation.  
   - Tools are implemented as callable async functions.

4. **Database Layer (PostgreSQL + pgvector)**  
   - Stores user trades, journals, embeddings, reviews, and configuration preferences.  
   - pgvector is used for storing and retrieving semantically encoded contexts (journal entries, insights, reflections).  

5. **Cache Layer (Redis)**  
   - Maintains ephemeral session data, conversation context, and quick retrieval caches for frequently used stats.  
   - Supports queueing for background tasks like report generation or trade classification.

6. **Storage Layer (R2/S3)**  
   - Stores files, screenshots, and attachments linked to trades or journals.  
   - Metadata is indexed in PostgreSQL for AI reference linking.

## 4. Context Flow

### 4.1 Input Pipeline

1. User interacts through AGUI (text, voice, or live editing).  
2. Co-PilotKit captures front-end state (active trade log, journal text, chart viewport).  
3. Context packet is built containing:  
   - Current page/module  
   - User preferences (tone, verbosity, focus)  
   - Relevant trade/journal data  
   - User embeddings (semantic history)
4. Context packet is streamed to Renata Runtime through WebSocket or REST endpoint.

### 4.2 Processing Flow

1. Renata.Core reads the packet, determines mode (Analyst / Coach / Mentor).  
2. Contextual filters load the appropriate tone and reasoning weights.  
3. Relevant embeddings are fetched from pgvector (top-N similarity).  
4. Tools execute computations (e.g., trade statistics, performance deltas).  
5. Reasoning layer composes structured response with contextual awareness.

### 4.3 Output

- Returns structured JSON response containing text output, action suggestions, and any tool results.  
- Co-PilotKit renders this in the AGUI environment (AI sidebar, inline notes, or voice response).

## 5. Mode Logic and Tone Mapping

Each mode uses the same reasoning engine but applies different tonal weights and response templates.

### 5.1 Analyst Mode
- **Goal:** Deliver raw, unfiltered performance truth.
- **Tone:** Clinical, direct, minimal emotion.
- **Behavior:** Cites hard stats, rules broken, quantifiable trends.
- **Prompt Skeleton:**
  ```
  Analyze {context}. Report only metrics, deviations, and data-driven explanations. Avoid motivational phrasing.
  ```

### 5.2 Coach Mode
- **Goal:** Encourage professional improvement without softening truth.
- **Tone:** Constructive, firm, forward-looking.
- **Behavior:** Frames observations as correctable actions.
- **Prompt Skeleton:**
  ```
  Analyze {context}. Identify the key weaknesses and propose structured improvement steps. Maintain professional tone.
  ```

### 5.3 Mentor Mode
- **Goal:** Build understanding through reflection and pattern recognition.
- **Tone:** Calm, reflective, analytical.
- **Behavior:** References both data and human patterns.
- **Prompt Skeleton:**
  ```
  Summarize {context}. Correlate emotional or behavioral themes with performance metrics. Encourage long-term awareness.
  ```

## 6. Memory and Learning

### 6.1 Memory Storage
- Each user interaction is semantically embedded and stored in pgvector.
- Embeddings represent trade logs, journal entries, and conversational summaries.
- Each embedding vector is tagged with metadata (timestamp, source type, performance context).

### 6.2 Retrieval
- On each request, Renata retrieves top 20 relevant embeddings per query.
- Vector similarity is computed using cosine distance.
- Aggregated context summary is dynamically appended to each prompt.

### 6.3 Reinforcement
- Renata learns user tone preferences by tracking feedback interactions (thumbs up/down or correction prompts).
- Over time, tone weightings are adjusted in the Core module’s config.

---

## 7. Security and Privacy

- All embeddings are per-user and encrypted at rest (AES-256 in pgvector storage).
- No global memory sharing; Renata cannot access data across users or workspaces.
- Sensitive data (account numbers, credentials) redacted pre-embedding.
- Co-PilotKit and AGUI streams are authenticated via signed JWT and expiring session tokens.

## 8. Example AI Flow

**User Action:**  
“Renata, run my weekly review.”

**Process:**  
1. AGUI captures active date range, journal summaries, and P&L data.  
2. Co-PilotKit packages context → API → Renata Runtime.  
3. Renata.Core selects **Coach Mode** (based on user preference).  
4. `analyze_performance()` tool calculates metrics and trends.  
5. Renata composes structured response:  
   - Text summary  
   - Key stats delta (Expectancy, Win %, R per Trade)  
   - Suggested focus areas

**Output:**
> “You averaged 0.82R per trade this week, down from 1.06R. Win rate improved 4%. Entries against trend produced the worst expectancy. Focus on alignment setups next week.”
