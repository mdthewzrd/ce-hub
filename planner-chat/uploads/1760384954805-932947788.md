# 🔄 Traderra — User Flows & Interaction Logic

## 1. Overview

This document defines every major user flow within **Traderra**, detailing the journey from login to daily reviews, journaling, and data analysis. It integrates **Renata’s conversational layer**, **AGUI state transitions**, and backend event triggers to form a seamless, adaptive experience.

Each flow is structured to reinforce Traderra’s values: **clarity, focus, and professionalism** — no gamification, no distraction.

---

## 2. Core User Flows

### 2.1 Login → Workspace → Dashboard

**Purpose:** Load user context, sync workspace, and display high-level metrics.

**Flow:**
1. User signs in via Clerk (OAuth or email).  
2. JWT token resolves to `workspace_id` → context injection.  
3. API loads workspace preferences, AI mode, and saved UI layout.  
4. Renata greets user contextually (e.g., “Good morning. Your session from yesterday closed +2.8R.”).  
5. Dashboard renders: Equity curve, Daily P&L, Key KPIs, Recent Journals.

**Triggers:**
- `GET /api/dashboard` — loads aggregates, last journal, review summary.
- `GET /api/preferences` — loads user settings, AI tone, stat visibility.

---

### 2.2 Dashboard → Daily Review Flow

**Purpose:** Conduct a structured performance review through Renata.

**Flow:**
1. User selects “Start Daily Review” (top right CTA or via AI command).  
2. API fetches trades tagged with today’s date.  
3. Renata begins analysis dialogue:
   > “You logged 12 trades today. Let’s review them by setup category.”
4. Data visualizations appear: Gross/Net P&L, Win %, Expectancy.  
5. Renata prompts reflection:
   > “You deviated from your entry rule twice. Would you like to note why?”
6. User responds verbally or types notes.  
7. Response saved to Journal → `POST /api/documents?kind=journal`.
8. Renata concludes:
   > “Summary: +3.4R net. Expectancy improved from last week. Review logged.”

**Outcomes:**
- Daily review created.  
- KPIs stored for weekly aggregation.  
- AI mode reinforced via feedback.

---

### 2.3 Journal Flow (Structured + Freeform)

**Purpose:** Allow user to document qualitative insights.

**Flow:**
1. User opens “Journal” tab or triggers “Open Journal” command.  
2. Chooses between templates (structured prompts) or freeform.  
3. Structured mode prompts appear sequentially:
   - Setup quality
   - Emotional state
   - Rule adherence
   - Lesson learned
4. Freeform mode uses single text input + semantic tagging (AI auto-labels phrases like *fear*, *early exit*, etc.).
5. When saved, journal content generates vector embedding → `POST /api/ai/embed`.  
6. Journal appears in “Recent Journals” panel with sentiment tag.

**Example Interaction:**
> User: “Felt impatient today and took early entries.”  
> Renata: “Logged as ‘Rule deviation: early entry’. I’ll monitor frequency.”

---

### 2.4 Trade Review Flow

**Purpose:** Deep-dive into trade-level performance.

**Flow:**
1. User selects trade card → opens modal with metrics.  
2. Data displayed:
   - Symbol, Side, Entry/Exit
   - Gross/Net P&L
   - R-multiple
   - Strategy reference
   - Linked journal note
3. Toggle controls:
   - Gross vs Net
   - % vs $ vs R (three-tier unit toggle)
4. Renata contextualizes:
   > “This trade deviated 0.5R from your expectancy baseline. Would you like to mark it as an outlier?”

**Actions:**
- Mark Outlier  → flags record.  
- Add Reflection → appends note.  
- Compare Setup → opens side-by-side analysis vs past trades.

---

### 2.5 Weekly Review Flow

**Purpose:** Aggregate daily reviews into higher-level insights.

**Flow:**
1. User triggers or receives scheduled prompt:
   > “Would you like to generate your weekly review?”
2. Worker compiles all trades/reviews for 7-day range.  
3. Metrics calculated:
   - Expectancy (gross/net)
   - Profit Factor
   - Win %
   - Total R
   - Equity curve summary
4. Renata presents:
   > “Weekly summary: +7.4R, Expectancy 0.84R. Consistency improved in setups B and D.”
5. AI offers deeper insight:
   > “Your best expectancy came from disciplined trade sizing. Do you want to explore risk variance?”
6. User confirms → opens trade distribution chart.

**Outcomes:**
- `review` document created.  
- Insights stored → accessible via `GET /api/reviews/:id`.

---

### 2.6 AI Query Flow (Ad-hoc Conversation)

**Purpose:** Enable flexible analysis or journaling queries.

**Examples:**
- “Show me all trades from last month where I broke my rules.”
- “Generate a summary of my performance since September.”
- “Analyze how my expectancy has evolved.”

**Flow:**
1. User types or speaks request.  
2. Renata parses → converts to structured query.  
3. `POST /api/ai/query` executes via internal tools.  
4. Response includes:
   - Natural language summary.
   - Structured data object for AGUI visualization.
5. Visualization updates instantly (chart/table).  
6. Renata may follow up contextually:
   > “Do you want to save this as a recurring insight?”

---

### 2.7 Export & Data Flow

**Purpose:** Allow traders to export or back up data easily.

**Flow:**
1. User clicks “Export Data”.  
2. Options: CSV, ZIP (All), PDF Summary.  
3. API bundles → generates signed URL.  
4. Renata confirms:
   > “Your data export is ready. It includes trades, reviews, and journals.”

## 3. Microflows

### 3.1 Gross/Net Toggle Logic
- Global state variable: `stats_mode = {gross|net}`.  
- All displayed KPIs, charts, and tooltips update in sync.  
- Auto-saved per-user preference.

### 3.2 Unit Toggle (%, $, R)
- Single top-right toggle group.  
- Propagates globally across dashboard and trade modals.  
- Renata acknowledges:
  > “Switched to R-multiple view. All metrics now relative to risk.”

### 3.3 AI Verbosity Adjustment
- Command: “Be more concise / detailed.”  
- Adjusts sentence length, response density, and chart complexity.

## 5. Adaptive Flow Logic (Renata)

| Scenario | AI Behavior |
|-----------|--------------|
| **Low performance streak** | Prioritizes analytical language (avoid motivation). |
| **High variance** | Focus on expectancy and process over outcome. |
| **New user** | Simplified UI and explanations. |
| **Returning pro** | Skip prompts; go straight to insights. |

---

## 6. AGUI Context-Sync Behaviors

### Overview
Renata and AGUI maintain bi-directional context synchronization:
- **From AI → UI:** Adjusts dashboards, opens panels, filters trades.
- **From UI → AI:** Every user action (click, filter, note) feeds context back into Renata’s runtime.

### Examples
| Trigger | Action |
|----------|--------|
| “Show my losing days” | Opens filtered chart view of negative P&L. |
| Toggle Gross → Net | Updates AGUI + confirms verbally. |
| Select trade | Loads trade modal + AI commentary. |

---

## 7. User Feedback & Adaptation

| Feedback Type | Result |
|----------------|--------|
| Thumbs Down | Reduces verbosity / rephrases future summaries. |
| “Tone too strict” | Switches sub-mode (Analyst → Softened). |
| “Be more detailed” | Expands next few answers by 20–30% tokens. |
| “Ignore this topic” | Adds filter tag to ignore contextually similar data. |

---

## 8. Error & Fallback Handling

| Failure | User Message |
|----------|----------------|
| API Timeout | “Connection issue. Retrying…” |
| No Trades Found | “No trades logged for that period.” |
| Data Mismatch | “Inconsistent fields detected. Would you like to re-validate the import?” |
| Unknown Command | “Not sure what you meant. Try asking in a different way.” |
