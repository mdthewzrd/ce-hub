# 🔌 Traderra — API Specification (v1)

> Audience: Backend/Frontend engineers, Claude Code agents, QA. REST-first with WebSocket events. Auth via Clerk JWT. All endpoints are **multi-tenant** and require `workspace_id` context.

---

## 0) Conventions

- **Base URL**: `https://api.traderra.com` (staging: `https://staging-api.traderra.com`)
- **Versioning**: Header `X-API-Version: 1` (URL `/v1` alias available).
- **Auth**: `Authorization: Bearer <clerk_jwt>`
- **Workspace**: `X-Workspace-Id: <uuid>` (validated against membership)
- **Content-Type**: `application/json`
- **Pagination**: `?limit=50&cursor=<token>`; response includes `{"items": [...], "next_cursor": "..."}`
- **Idempotency**: Optional header `Idempotency-Key: <uuid>` for POST/PUT.
- **Rate limits**: `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `Retry-After` on 429.
- **Errors**: JSON `{ "error": { "code": "string", "message": "string", "details": object } }` with standard HTTP codes.

## 1) Auth & Identity

### 1.1 GET /v1/me
Returns current user & memberships.

**Response**
```json
{
  "user": {"id":"uuid","email":"user@x.com","clerk_user_id":".."},
  "memberships": [{"workspace_id":"uuid","role":"owner"}]
}
```

### 1.2 GET /v1/workspaces
List workspaces for the user.

### 1.3 POST /v1/workspaces
Create a workspace (owner role).

**Body**
```json
{"name":"Acme Trading"}
```

## 3) Trades & Executions

### 3.1 GET /v1/trades
Query trades with filters.

**Query**: `from`, `to`, `symbol`, `side`, `strategy_id`, `tags`, `account_id`, `cursor`, `limit`

### 3.2 POST /v1/trades
Create or upsert a trade.

**Body**
```json
{
  "trade_account_id":"uuid",
  "symbol":"AAPL",
  "side":"long",
  "opened_at":"2025-08-01T14:31:00Z",
  "closed_at":"2025-08-01T15:18:00Z",
  "qty":100,
  "avg_entry_price":192.15,
  "avg_exit_price":193.42,
  "pnl":127.0,
  "tags":["breakout"]
}
```

### 3.3 GET /v1/trades/{id}
Returns trade with executions and links.

### 3.4 PUT /v1/trades/{id}
Edit trade fields.

### 3.5 DELETE /v1/trades/{id}
Soft delete.

### 3.6 Executions
- `GET /v1/trades/{id}/executions`
- `POST /v1/trades/{id}/executions` → `{ ts, side, price, qty, fee }`
- `DELETE /v1/executions/{exec_id}`

## 5) Imports (CSV & Broker)

### 5.1 POST /v1/imports/csv
Initiate CSV import; returns `batch_id` and S3 pre-signed URL or direct upload.

**Response**
```json
{"batch_id":"uuid","upload_url":"https://..."}
```

### 5.2 POST /v1/imports/{batch_id}/validate
Validate/preview mapping (auto-detect columns per source).

**Response**
```json
{
  "rows_total": 1248,
  "rows_ok": 1232,
  "warnings": ["2 rows missing timezone"],
  "column_map": {"Symbol":"symbol","Time":"ts","Action":"side","Price":"price","Qty":"qty"}
}
```

### 5.3 POST /v1/imports/{batch_id}/apply
Normalize and write rows → trades/executions. Idempotent.

### 5.4 GET /v1/imports/{batch_id}
Status, errors, and summary counts.

```json
  "unit":"R",

    "avg_r_winner": 1.65

```

### 8.2 GET /v1/stats/equity

### 8.3 GET /v1/stats/pnl-bars

### 8.4 GET /v1/stats/distribution

### 8.5 GET /v1/stats/grouped
`by=instrument|weekday|hour|setup` → grouped metrics.

## 13) WebSocket Events

**URL**: `wss://api.traderra.com/v1/ws`  
**Auth**: Same JWT; `X-Workspace-Id` header or query param.

### 13.1 Event Types
- `ai.message` — Renata responses/stream tokens
- `review.completed` — background review finished
- `import.progress` — CSV/broker import status
- `sync.completed` — Notion/Drive sync done
- `stats.updated` — KPIs recalculated for current filters

**Message Envelope**
```json
{ "type":"ai.message", "ts":"2025-10-08T12:01:22Z", "data":{...} }
```

## 14) Errors & Codes

| HTTP | code | meaning |
|------|------|---------|
| 400 | `bad_request` | Invalid input |
| 401 | `unauthorized` | Missing/invalid JWT |
| 403 | `forbidden` | No workspace access |
| 404 | `not_found` | Resource missing |
| 409 | `conflict` | Idempotency or unique constraint |
| 422 | `validation_error` | Schema validation failed |
| 429 | `rate_limited` | Slow down |
| 500 | `internal_error` | Unexpected server error |

## 16) Security

- All endpoints enforce RLS via `workspace_id` context.  
- OAuth tokens stored encrypted; never returned raw.  
- Audit trail on mutating endpoints (who/when/what).  
- Webhooks signed with HMAC; replay protection via nonce + timestamp window.

## 17) OpenAPI & SDKs

- **OpenAPI** served at `/v1/openapi.json`.
